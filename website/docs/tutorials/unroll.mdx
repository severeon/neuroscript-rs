# Unroll

The `unroll` construct expands repeated layers at **compile time**. Instead of writing the same neuron call N times, you write it once and let the compiler generate the repetitions. Each iteration gets its own independent weights by default.

## Why "Unroll"?

The name is intentional — this is not a runtime loop. The compiler literally unrolls the construct into N separate graph nodes before code generation. The count must be a literal integer or a parameter with a default value, never a runtime value.

## Sequential Threading

The most common form chains iterations sequentially — the output of iteration *k* feeds the input of iteration *k+1*:

<InteractiveExample
  title="Transformer Stack"
  description="N transformer blocks chained sequentially via unroll"
  initialCode={`neuron TransformerStack(d_model, num_heads, d_ff, num_layers=6):
    in: [batch, seq, d_model]
    out: [batch, seq, d_model]
    graph:
        in ->
            unroll(num_layers): ->
                TransformerBlock(d_model, num_heads, d_ff)
            out`}
/>

The `->` after the colon is what makes this sequential. It means "thread the data through each iteration." The unrolled block must have compatible input/output shapes since each output becomes the next input.

## Context-Scoped Bindings

When you need named bindings for each iteration, declare them inside an `unroll` block in the `context:` section:

<InteractiveExample
  title="Context-Scoped Unroll"
  description="Each iteration gets its own independent block instance"
  initialCode={`neuron NamedStack(d_model, num_heads, d_ff, num_layers=3):
    in: [batch, seq, d_model]
    out: [batch, seq, d_model]
    context:
        unroll(num_layers):
            block = TransformerBlock(d_model, num_heads, d_ff)
    graph:
        in ->
            unroll(num_layers): ->
                block
            out`}
/>

The compiler matches context-unroll bindings to graph-unroll references by the unroll parameter. Iteration 0 gets instance 0, iteration 1 gets instance 1, and so on. Each `block` is an independent module with its own weights.

## Weight Sharing with @static

To reuse the **same weights** across all iterations (like a Universal Transformer), combine `unroll` with `@static`:

<InteractiveExample
  title="Weight Sharing"
  description="@static makes all iterations share one set of weights"
  initialCode={`neuron UniversalTransformer(d_model, num_heads, d_ff, num_steps=6):
    in: [batch, seq, d_model]
    out: [batch, seq, d_model]
    context:
        unroll(num_steps):
            @static block = TransformerBlock(d_model, num_heads, d_ff)
    graph:
        in ->
            unroll(num_steps): ->
                block
            out`}
/>

`@static` means one instance, reused across all references. The generated code creates a single `TransformerBlock` in `__init__` and calls it `num_steps` times in `forward`.

## End-to-End Example: GPT-2

`unroll` composes naturally with other pipeline steps — it's just another block in the chain:

<InteractiveExample
  title="GPT-2 Small"
  description="Full model: embedding → unrolled transformer blocks → head"
  initialCode={`neuron GPT2Small(vocab_size=50257, d_model=768, num_heads=12, d_ff=3072, num_layers=12):
    in: [*, seq]
    out: [*, seq, vocab_size]
    context:
        embed = Embedding(vocab_size, d_model)
        unroll(num_layers):
            block = TransformerBlock(d_model, num_heads, d_ff)
        ln_f = LayerNorm(d_model)
        head = Linear(d_model, vocab_size)
    graph:
        in -> embed -> e
        e ->
            unroll(num_layers): ->
                block
            mid
        mid -> ln_f -> head -> out`}
/>

This replaces what would otherwise be 12 hand-written `TransformerBlock` calls.

## Multiple Bindings Per Iteration

Each unroll iteration can contain multiple bindings:

```neuroscript
neuron MultiScaleEncoder(dim, num_heads, d_ff, num_scales=4):
    in: [*, seq, dim]
    out: [*, seq, dim]
    context:
        unroll(num_scales):
            encoder = TransformerBlock(dim, num_heads, d_ff)
            norm = LayerNorm(dim)
    graph:
        in ->
            unroll(num_scales): ->
                encoder
                norm
            out
```

Each iteration gets its own independent `encoder` and `norm` pair.

## Mixed Scoping

Unroll-scoped and regular bindings compose freely:

```neuroscript
neuron HybridStack(dim, num_heads, d_ff, num_layers=6):
    in: [*, seq, dim]
    out: [*, seq, dim]
    context:
        @static shared_norm = LayerNorm(dim)
        unroll(num_layers):
            block = TransformerBlock(dim, num_heads, d_ff)
    graph:
        in ->
            unroll(num_layers): ->
                block
                shared_norm
            out
```

Each iteration gets its own `block` (independent weights), but all iterations share the same `shared_norm` instance.

## Key Rules

- The count must be a **literal integer or a parameter with a default value** — it must be resolvable at compile time
- The `->` after the colon means **sequential threading** (output feeds next input)
- Each iteration gets **independent weights** unless `@static` is used
- Context-unroll bindings are **scoped** — they can only be referenced inside a matching graph-unroll
- Referencing an unroll-scoped binding outside a matching unroll is a **compile error**
